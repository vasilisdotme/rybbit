## Core Web Vitals

The performance endpoints return data for the following Core Web Vitals metrics:

| Metric | Description | Good | Needs Improvement | Poor |
|--------|-------------|------|-------------------|------|
| **LCP** | Largest Contentful Paint - loading performance | < 2.5s | 2.5s - 4s | > 4s |
| **CLS** | Cumulative Layout Shift - visual stability | < 0.1 | 0.1 - 0.25 | > 0.25 |
| **INP** | Interaction to Next Paint - interactivity | < 200ms | 200ms - 500ms | > 500ms |
| **FCP** | First Contentful Paint - initial render | < 1.8s | 1.8s - 3s | > 3s |
| **TTFB** | Time to First Byte - server response time | < 800ms | 800ms - 1.8s | > 1.8s |

## Get Performance Time Series

<div className="grid grid-cols-1 xl:grid-cols-2 gap-8">
<div>

```
GET /api/performance/time-series/:site
```

Returns performance metrics over time, broken down by configurable time buckets. Useful for charting performance trends.

### Path Parameters

<TypeTable
  type={{
    site: {
      description: 'Site ID or site identifier',
      type: 'string | number',
      required: true,
    },
  }}
/>

### Query Parameters

Accepts all [Common Parameters](/docs/api/getting-started#common-parameters) plus the following:

<TypeTable
  type={{
    bucket: {
      description: 'Time bucket size for grouping data',
      type: 'TimeBucket',
      typeDescription: '"minute" | "five_minutes" | "hour" | "day" | "week" | "month"',
      required: true,
    },
  }}
/>

### Response

<TypeTable
  type={{
    data: {
      description: 'Array of time-bucketed performance data',
      type: 'PerformancePoint[]',
    },
  }}
/>

#### PerformancePoint Object

Each point includes all percentiles for each metric plus:

<TypeTable
  type={{
    time: {
      description: 'Start of the time bucket',
      type: 'string',
      typeDescription: 'ISO 8601 datetime',
    },
    event_count: {
      description: 'Number of performance events in this bucket',
      type: 'number',
    },
  }}
/>

## Get Performance by Dimension

<div className="grid grid-cols-1 xl:grid-cols-2 gap-8">
<div>

```
GET /api/performance/by-dimension/:site
```

Returns performance metrics broken down by a specific dimension (page, country, browser, etc.). Useful for identifying performance issues on specific pages or for specific user segments.

### Path Parameters

<TypeTable
  type={{
    site: {
      description: 'Site ID or site identifier',
      type: 'string | number',
      required: true,
    },
  }}
/>

### Query Parameters

Accepts all [Common Parameters](/docs/api/getting-started#common-parameters) plus the following:

<TypeTable
  type={{
    dimension: {
      description: 'Dimension to break down by',
      type: 'string',
      typeDescription: '"pathname" | "country" | "device_type" | "browser" | "operating_system" | "region"',
      required: true,
    },
    page: {
      description: 'Page number (1-indexed)',
      type: 'number',
      required: false,
    },
    limit: {
      description: 'Number of results per page',
      type: 'number',
      default: 100,
    },
    sort_by: {
      description: 'Field to sort by',
      type: 'string',
      default: 'event_count',
    },
    sort_order: {
      description: 'Sort direction',
      type: 'string',
      typeDescription: '"asc" | "desc"',
      default: 'desc',
    },
  }}
/>

### Response

<TypeTable
  type={{
    data: {
      description: 'Paginated response object',
      type: 'PaginatedResponse',
    },
  }}
/>

#### DimensionPerformance Object

Each object includes the dimension value plus all percentiles:

<TypeTable
  type={{
    pathname: {
      description: 'The dimension value (field name matches dimension param)',
      type: 'string',

    event_count: {
      description: 'Number of performance events',
      type: 'number',
    },
    lcp_avg: {
      description: 'Average LCP',
      type: 'number',

    lcp_p50: {
      description: 'LCP 50th percentile',
      type: 'number',
    },
    lcp_p75: {
      description: 'LCP 75th percentile',
      type: 'number',
    },
    lcp_p90: {
      description: 'LCP 90th percentile',
      type: 'number',
    },
    lcp_p99: {
      description: 'LCP 99th percentile',
      type: 'number',
    },
  }}
/>
